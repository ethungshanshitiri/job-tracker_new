# Faculty Job Tracker — IITs & NITs

A static website that tracks current **Assistant Professor** and **Associate Professor** openings across all IITs and NITs in India. Data refreshes automatically every 12 hours via GitHub Actions and deploys to GitHub Pages.

---

## What the site does

The scraper starts at each institute's homepage and crawls up to three link-hops deep, staying within the institute's own domain. It identifies pages that look like faculty recruitment listings, reads the full text of those pages, and applies strict inclusion and exclusion filters. The frontend displays only institutes where confirmed, current openings were found.

---

## How the crawler works

For each institute, a breadth-first crawl begins at the homepage. Every link found on a page is scored for recruitment relevance based on its URL path and anchor text. Links that score zero or below are discarded. The most promising links are visited first. The crawl stops when 40 pages have been visited for that institute or the depth limit of three hops is reached.

When a page is fetched, it is read in its entirety as plain text (HTML tags stripped, entities decoded). The page then passes through a five-step pipeline.

**Step 1 — Quick gate.** The page must contain at least one of the target rank terms (assistant professor, associate professor) and either an inclusion phrase or a target department term. Pages that fail this gate are discarded without further processing but their links may still be enqueued if the crawl is not yet at max depth.

**Step 2 — Exclusion dominance check.** If exclusion terms appear heavily (research assistant, JRF, SRF, postdoctoral, non-teaching, etc.) and no genuine faculty inclusion phrase is present alongside rank terms, the page is suppressed. A page with rank terms and three or more exclusion terms but zero inclusion phrases is treated as a false positive.

**Step 3 — Closed signal check.** If the page contains language such as "last date expired", "recruitment closed", "position filled", or "applications closed", it is suppressed entirely.

**Step 4 — Rank and department detection.** The plain text is scanned for all rank terms and all department synonyms. Confirmed results require at least one rank term and at least one target department to be present somewhere on the same page.

**Step 5 — Deadline and mode extraction.** The scraper attempts to extract a future deadline by matching common Indian date formats (DD/MM/YYYY, DD Month YYYY, YYYY-MM-DD, Month DD YYYY). Rolling-basis language is also detected. Confidence is scored based on the combination of signals found.

## Inclusion and exclusion rules

**Included ranks only:**
- Assistant Professor
- Associate Professor

**Included department families only:**
- Electrical Engineering (and common naming variants)
- Electronics and Communication Engineering (and variants, including ECE)
- Computer Science and Engineering (and variants, including CSE)

**Never shown:**
- Research Assistant, Project Assistant, JRF, SRF
- Postdoctoral positions of any kind
- Visiting, ad hoc, or contract faculty
- Lecturers and Teaching Assistants
- Non-teaching, administrative, technician, scientist, officer roles
- PhD admission notices
- General staff recruitment

The full keyword lists are in `config/sources.json` under `inclusion_keywords`, `exclusion_keywords`, and `closed_signals`.

## How to add or update an institute

Open `config/sources.json` and find the institute under `iits` or `nits`. The only field that needs to be correct is `homepage` — the crawler discovers recruitment pages from there automatically. Set `homepage` to the institute's main website root (e.g. `https://www.example.ac.in`).

To add a new department synonym, add it to the relevant array inside `target_departments`. No code changes are needed; the next scraper run picks it up.

To tune crawl behavior (depth, page limit, delay), edit the `crawler` block in `config/sources.json`.

## How to run locally

Requirements: Node.js 18 or later.

```bash
git clone https://github.com/<your-org>/job-tracker.git
cd job-tracker

# Run the crawler (writes to data/jobs.json)
# Warning: this makes live HTTP requests to ~53 institute servers and takes 10-20 minutes
node scripts/scrape.js

# Serve the site locally
python3 -m http.server 8000
# Open http://localhost:8000
```

## Deployment

The GitHub Actions workflow in `.github/workflows/refresh-and-deploy.yml` runs on a schedule (00:30 and 12:30 UTC) and also supports manual dispatch.

One-time setup required before the first deploy: in your repository's Settings, go to **Pages** and set the source to **GitHub Actions**. After that, every workflow run deploys the updated site automatically.

To trigger a manual refresh go to **Actions → Refresh Jobs & Deploy → Run workflow**.

## Known limitations

**Anti-bot measures.** Some institute servers may rate-limit or block automated requests. If a page fetch fails it is logged as a warning and the crawl continues. Check the Actions run log for `[fetch-err]` lines.

**PDF job advertisements.** The scraper does not read PDF files. If an institute publishes its recruitment notice only as a PDF linked from its recruitment page, that advertisement will not appear in results. The page linking to the PDF will still be found, but if the page itself contains no rank or department text, it will be skipped.

**JavaScript-rendered pages.** The scraper fetches raw HTML only. Pages that load job listings via JavaScript after page load (single-page applications, dynamic tables) will not have their listings captured. The page may still be found but the listings will be invisible to the scraper.

**Deadline parsing.** The parser handles the most common Indian academic date formats. Unusual formats or dates written in narrative form ("applications close at the end of May") will not be extracted. The status will show "See official page" in those cases.

**Crawl scope cap.** Each institute is limited to 40 pages per run. Institutes with very large or deeply nested websites may not have all recruitment subpages reached.

**Data freshness.** The frontend shows a warning banner if `data/jobs.json` is older than 25 hours or empty.

## Disclaimer

This site is a convenience index only. Always verify the full details on the official institute recruitment page before applying. Deadlines, departments, and ranks shown here may lag behind official updates.
